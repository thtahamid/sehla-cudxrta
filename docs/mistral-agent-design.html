Autonomous Congestion Agent - design spec

Powered by Mistral Agents API
Phase 1 · Design

An agent that watches the corridor and reasons about why it's breaking down.

A propose-and-approve autonomous agent for the City Walk traffic command center. It monitors the managed corridors, detects degradation, and drafts a signal recommendation - with a reasoning trace and an expected-impact estimate. The operator stays in command: nothing changes until they approve.

The only file that talks to Mistral. Get-or-create agent (cached), conversation lifecycle, forced advisory tool, schema validation. SDK-or-REST isolated here.

Source of truth stays on the client - it only sends a compact snapshot and renders the advisory. No business logic moves server-side.
03

The advisory contract

The agent is forced to call one tool. Its arguments are the advisory - always structured, always validated.

// emit_advisory - the single forced function tool
{
  severity: "info" | "watch" | "act",
  corridor: string,            // e.g. "Al Badaa / 83b"
  cause: string,               // the reasoning: WHY it is degrading
  recommendation: {
    action: "set_signal_strategy" | "set_scenario" | "focus_intersection" | "monitor",
    params: { strategy?, scenario?, intersectionId? }
  },
  expectedImpact: { queueDeltaPct: number, delayDeltaSec: number },
  confidence: number,          // 0..1
  rationale: string            // 1-2 sentence operator-facing summary
}
Closed action enum. Every action maps to something the app can already execute, so Approve is a safe, deterministic dispatch - never free-form control.
monitor = no-op. When the agent decides not to act, it still returns a structured advisory explaining why it is holding.

06

Resilience

Mirrors the existing TomTom fallback pattern - the dashboard is never blocked by AI.

○
No MISTRAL_API_KEY
Route returns { mode: "disabled" }. Rail shows a quiet "copilot not configured" hint. App fully usable.
⏱
Timeout / API error (8s)
Typed error. Rail shows "couldn't analyze, retry". Never crashes the dashboard.
⟳
Schema-invalid tool args
One automatic retry, then a graceful fail state.
↧
429 / 5xx on primary model
Falls back Medium 3.5 → Small 4 before surfacing an error.
