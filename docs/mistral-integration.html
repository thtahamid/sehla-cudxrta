Instructions (abridged): On each turn, receive a JSON snapshot of the live managed corridor and call emit_advisory exactly once. Diagnose why a corridor is degrading (spillback, fixed-time phasing, downstream blocking). Recommend at most one executable action; only propose green-wave when the strategy is still standard and there is a queue-and-delay-backed win, else monitor. Do not repeat an advisory already emitted this conversation unless conditions materially changed.
04

How the memory works

The key discovery: the operator's decision becomes the tool's function result.

App → Server
Congestion snapshot (queues, delays, phases) for the managed corridor.
Server → Mistral
POST /v1/conversations  { agent_id, inputs: snapshot }
Mistral → Server
Returns conversation_id + a function.call to emit_advisory with a tool_call_id. The args ARE the advisory.
Operator
Approve applies green-wave in the app; queue drains. The decision is remembered as pending.
Server → Mistral
Next pass: POST /v1/conversations/{id} with a function.result for that tool_call_id ({decision:"approved"}) then the new snapshot. this is the memory
Mistral → Server
Sees its advice was applied and the corridor recovered → emits monitor instead of repeating green-wave.

Without answering the pending tool call, the API returns 400 "Cannot append other inputs while function results are still missing". Turning that constraint into the feature is what makes the agent stateful and non-repetitive.

07

Three findings that shaped the build

67s

Reasoning model was too slow

magistral-medium gave good diagnoses but took ~67s per conversation turn - unusable interactively.

Fix → PATCH the agent to mistral-medium-latest (1-3s), keep magistral noted for offline analysis.
enum

The model can drift on values

A forced tool call once returned strategy:"congestion" - not a value the app can execute.

Fix → strict enums in the schema + a server-side validator with one retry, then fallback.
400

Conversations are turn-locked

You cannot append a new snapshot while a tool call is unanswered - the API 400s.

Fix → make the operator's Approve/Dismiss the function result. Constraint becomes the memory.
